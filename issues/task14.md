# Task14: /ndb-renewalのアクセシビリティとUI品質をClaude Codeで改善する

## 目的

Task13で実装した `/ndb-renewal` ページを、アクセシビリティ、レスポンシブ、保守性の観点で改善する。

このタスクは時間が余った人向けの発展課題です。見た目を整えるだけでなく、実務でレビューに出せるページに近づけるための品質確認を体験します。

## 所要時間

約20-30分

## 前提条件

- Task13が完了していること
- `/ndb-renewal` ページが表示できること
- `npm run lint` と `npm run test` を実行できること
- Claude Codeに「まずレビューだけ」「Highだけ修正」など、変更範囲を指定できること

## このタスクで学ぶこと

- アクセシビリティ観点でUIをレビューする方法
- セマンティックHTMLと見出し構造の確認方法
- リンクとボタンの使い分け
- レスポンシブ表示の確認方法
- Claude Codeに品質観点を指定してレビューさせる方法
- レビュー指摘を重要度で分け、修正範囲を制御する方法

## ミッション

Task13で作成した `/ndb-renewal` ページを、以下の観点でレビューし、重要度の高いものから改善してください。

- セマンティックHTML
- 見出し構造
- リンクとボタンの意味
- CTAの文言
- alt属性や画像利用
- カラーコントラスト
- スマートフォン表示
- キーボード操作
- コンポーネントの読みやすさ

## 注意点

- まず修正せず、レビュー結果を出してください
- すべての指摘を一度に直そうとしないでください
- 重要度Highの指摘を優先してください
- 見た目の好みだけで大きく作り替えないでください
- 会社情報の意味を変える文言変更は避けてください
- 架空の数字、実績、取引先、制度を追加しないでください

## 実装の進め方

### ステップ1: 現在の実装を確認する

Claude Codeに依頼:

```text
Task14を進めます。

/ndb-renewal ページの現在の実装を確認してください。
まず修正せず、以下を説明してください。

1. 関連ファイル
2. ページ構成
3. コンポーネント構成
4. テストの有無
5. UI品質を確認する時に見るべきポイント

まだファイルは変更しないでください。
```

### ステップ2: アクセシビリティとUI品質をレビューする

Claude Codeに依頼:

```text
/ndb-renewal ページをアクセシビリティとUI品質の観点でレビューしてください。

観点:
1. セマンティックHTMLになっているか
2. h1, h2, h3 の見出し構造が自然か
3. ボタンとリンクの使い分けが適切か
4. CTAの文言が分かりやすいか
5. スマートフォン幅で崩れそうな箇所がないか
6. カラーコントラストに問題がないか
7. キーボード操作で困る箇所がないか
8. ジュニアエンジニアが保守しやすい構成か

まず修正せず、High / Medium / Low に分けて指摘してください。
```

### ステップ3: High指摘だけを修正する

Claude Codeのレビュー結果を確認し、High指摘だけを修正します。

Claude Codeに依頼:

```text
レビュー結果のうち、重要度Highのものだけ修正してください。

条件:
- 変更対象を最小限にしてください
- 会社情報の意味は変えないでください
- 架空情報は追加しないでください
- 修正後、どのファイルをどう変更したか説明してください
```

### ステップ4: 表示と操作を確認する

以下を確認してください。

- PC幅で見出しとセクションの流れが自然
- スマートフォン幅でテキストやCTAがはみ出さない
- CTAがリンクまたはボタンとして自然
- キーボード操作で主要なリンクに移動できる
- 画像を使っている場合、alt属性が適切

### ステップ5: テストとlintを実行する

```bash
cd example1
npm run lint
npm run test
```

必要に応じて、Task13で追加したテストを更新してください。
