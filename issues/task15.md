# Task15: /ndb-renewal開発で得た知見をCLAUDE.mdとSkillに反映する

## 目的

Task13とTask14で行った実装、テスト、レビューの流れを、次回以降も再利用できる形に整理する。

Claude Codeに毎回長い指示を出すのではなく、プロジェクトのルールやレビュー観点を `CLAUDE.md` やSkillに残すことで、チーム開発に近いAI活用を体験します。

このタスクは時間が余った人向けの発展課題です。

## 所要時間

約20-30分

## 前提条件

- Task13が完了していること
- 可能であればTask14も完了していること
- `CLAUDE.md` の役割を理解していること
- `.claude/skills/` の基本構成を理解していること
- Claude Codeに「まず提案だけ」「確認後に編集」と依頼できること

## このタスクで学ぶこと

- 実装で得た学びをプロジェクトルールへ反映する方法
- 会社情報を扱う時の安全なルール化
- Claude Code向けのレビュー観点を整備する方法
- Skillを使って繰り返し作業を短い指示で呼び出す方法
- AI活用を個人技ではなくチーム運用に近づける方法

## ミッション

`/ndb-renewal` の実装とレビューで得た知見をもとに、Claude Codeが今後このプロジェクトでより安全に作業できるように、以下を改善してください。

- `CLAUDE.md`
- 必要に応じて `.claude/skills/.../SKILL.md`
- 必要に応じて `.claude/agents/...`

## 追加・改善する観点

### CLAUDE.mdに追記する候補

- `/ndb-renewal` は研修用モックであり、本番サイトではないこと
- 既存サイトのHTML / CSS / JavaScriptをコピーしないこと
- 会社情報を扱う場合は、架空の数字、実績、取引先、制度を追加しないこと
- 実装前に調査、計画、変更対象を出すこと
- UI実装ではレスポンシブとアクセシビリティを確認すること
- テスト追加時は、ユーザーから見える文言と導線を中心に検証すること
- ジュニアエンジニアが読みやすいコードにすること

### Skillにする候補

- `corporate-page-review`
  - 会社情報を扱うページのレビュー用Skill
- `ui-accessibility-review`
  - UIとアクセシビリティのレビュー用Skill
- `safe-public-info-writing`
  - 公開情報を扱う時に架空情報を混ぜないための文言レビューSkill

## 注意点

- まず修正せず、追記案やSkill案だけを出してください
- `CLAUDE.md` を長くしすぎないでください
- 具体的な学びだけを残してください
- 既存の研修進行ルールと矛盾しないようにしてください
- Skillを作る場合は、名前、目的、発火条件、レビュー観点を明確にしてください
- 必須ではありません。時間が足りない場合は、`CLAUDE.md` の追記案だけで完了として構いません

## 実装の進め方

### ステップ1: Task13とTask14の学びを整理する

Claude Codeに依頼:

```text
Task15を進めます。

Task13とTask14で /ndb-renewal を実装・レビューした内容を踏まえて、
今後Claude Codeがこのプロジェクトでより安全に作業できるように、
CLAUDE.mdへ追記すべきルールを提案してください。

観点:
- 会社情報を扱う時の注意点
- 架空情報を追加しないためのルール
- 実装前に調査・計画を出すルール
- UI実装時のレビュー観点
- テスト方針
- ジュニアエンジニアが読みやすいコードにするための方針

まず修正せず、追記案だけ出してください。
```

### ステップ2: CLAUDE.mdに追記する

提案を確認し、必要なものだけ `CLAUDE.md` に反映します。

Claude Codeに依頼:

```text
提案のうち、この研修用リポジトリに残す価値が高いものだけをCLAUDE.mdに追記してください。

条件:
- 既存の研修進行ルールと重複しすぎないこと
- 長くしすぎないこと
- /ndb-renewal が研修用モックであることを明記すること
- 会社情報を扱う時に架空情報を追加しないルールを入れること
```

### ステップ3: Skill化するか判断する

時間がある場合は、レビュー観点をSkillにします。

Skillを作る場合の候補:

```text
Skill名:
corporate-page-review

目的:
公開情報をもとに作成したコーポレートサイト改善モックをレビューする。

レビュー観点:
- 架空の数字、実績、取引先、制度が混ざっていないか
- 会社情報の意味を勝手に変えていないか
- 信頼感のある文言になっているか
- CTAが自然か
- Next.js / Reactの実装として保守しやすいか
- アクセシビリティとレスポンシブに問題がないか
- テストが不足していないか
```

Claude Codeに依頼:

```text
会社情報を扱うページをレビューするためのSkillを作成してください。

Skill名:
corporate-page-review

目的:
公開情報をもとに作成したコーポレートサイト改善モックをレビューする。

まず既存の .claude/skills の構成を確認し、同じ形式で作成してください。
作成前に、作成ファイルと内容の概要を説明してください。
```

### ステップ4: 変更内容を確認する

```bash
git diff
```

確認ポイント:

- `CLAUDE.md` が長くなりすぎていない
- 会社情報を扱う時の注意点が明確
- 研修用モックと本番サイトの区別が明確
- Skillを作った場合、既存Skillと役割が重複しすぎていない

### ステップ5: Claude Codeに新しいルールを説明させる

Claude Codeに依頼:

```text
今回追加したCLAUDE.mdまたはSkillの内容を読んで、
次回 /ndb-renewal のような会社情報を扱うページを実装・レビューする時に、
Claude Codeが守るべきルールを説明してください。
```
