# Task13: 日本デェイブレイク株式会社のコーポレートサイト改善モックをClaude Codeで実装する

## 目的

Claude Codeを使って、実務に近い開発フローを一通り体験する。

今回は日本デェイブレイク株式会社の現行コーポレートサイトを参考に、既存のNext.jsプロジェクト内へ研修用のトップページ改善モックを作成する。

## 所要時間

約2時間

## 前提条件

- Task 1-12が完了していること
- `example1` のNext.jsアプリが動作すること
- Vitestの基本操作を理解していること
- Claude Codeに「調査」「計画」「実装」「レビュー」を分けて依頼できること

## このタスクで学ぶこと

- AIにいきなり実装させず、先に調査と計画をさせる進め方
- 公開情報をもとにした安全なリニューアルモックの作り方
- Next.js App Routerでの新規ページ作成
- コンポーネント分割とレスポンシブ対応
- Vitestで画面仕様を固定する方法
- Skill / Sub-agentを使った実装後レビュー

## ミッション

日本デェイブレイク株式会社の現行コーポレートサイトを参考に、研修用のトップページ改善モックをNext.jsで実装してください。

既存サイトのソースコードは使用しません。公開されている会社情報とサービス情報だけを参考にして、`/ndb-renewal` パスに新しいページを作成します。

## 参考情報

現行サイト:

https://ndbnet.co.jp/index.html

現行サイトから読み取れる主な情報:

- メインメッセージ
  - 「お客様と共に、情報技術で未来を創る。」
- サービス
  - 開発・導入サービス
  - オンサイトサービス
  - コンサルテーションサービス
- コンテンツ導線
  - 会社案内
  - 採用情報
  - お問い合わせ
- News

## 実装対象

以下のパスでページを作成してください。

```text
/ndb-renewal
```

想定ファイル:

```text
example1/src/app/ndb-renewal/page.tsx
example1/src/app/ndb-renewal/page.test.tsx
example1/src/app/ndb-renewal/_components/...
example1/src/app/ndb-renewal/page.module.css
```

Next.js App Routerでは、`app/ndb-renewal/page.tsx` が `/ndb-renewal` のページになります。

## 要件

### 1. まず調査と設計を行う

いきなりコードを変更せず、Claude Codeに以下を整理させてください。

- 現行サイトから読み取れる主要情報
- `/ndb-renewal` に必要なセクション
- 改善できそうなUX / UI上のポイント
- コンポーネント分割案
- 実装対象ファイル
- テスト方針
- 架空情報を追加しないための注意点

### 2. ページに含めるセクション

最低限、以下を含めてください。

- ヒーローセクション
  - キャッチコピー
  - サブコピー
  - CTAボタン
- サービス紹介
  - 開発・導入サービス
  - オンサイトサービス
  - コンサルテーションサービス
- 会社の特徴
  - 情報システムのライフサイクル全般を支援すること
  - 顧客先での支援があること
  - 業務ノウハウやプロジェクト管理経験を活かした支援
- 採用向けセクション
  - ジュニアエンジニアにも伝わる会社紹介
- Newsセクション
- お問い合わせ導線

### 3. 制約

- 既存サイトのHTML / CSS / JavaScriptはコピーしない
- 公開情報を参考にした研修用モックとして実装する
- 架空の数字、実績、取引先、制度を追加しない
- 会社情報の意味を勝手に変えない
- デザインは現代的にしてよいが、信頼感を優先する
- レスポンシブ対応する
- コンポーネントを適切に分割する
- 日本語の文言にする

### 4. テスト

最低限、以下をテストしてください。

- `/ndb-renewal` ページが表示される
- ヒーローの見出しが表示される
- サービス3件が表示される
- 採用向けセクションが表示される
- お問い合わせ導線が表示される

テスト方法は、既存のVitest設定に合わせてください。

追加依存を避けたい場合は、`react-dom/server` の `renderToStaticMarkup` でページHTMLを検証できます。より実践的にテストしたい場合は、`@testing-library/react` と `@testing-library/jest-dom` を追加して画面表示を検証してください。

`vitest.config.ts` のcoverage対象がAPIだけになっている場合、今回のページをcoverageに含めるかどうかもClaude Codeに確認してください。

## 実装の進め方

### ステップ1: 現行サイトの情報を整理する

Claude Codeに依頼:

```text
日本デェイブレイク株式会社のコーポレートサイト改善モックを作ります。

既存サイトのソースコードは使わず、公開情報だけを参考にして、
このNext.jsプロジェクト内に /ndb-renewal ページを作成したいです。

まず実装せず、以下を整理してください。

1. 現行サイトから読み取れる主要情報
2. /ndb-renewal に必要なセクション
3. 改善できそうなUX / UI上のポイント
4. コンポーネント分割案
5. 実装対象ファイル
6. テスト方針
7. 架空情報を追加しないための注意点

まだコードは変更しないでください。
```

### ステップ2: 実装計画を確認する

Claude Codeの提案をそのまま採用せず、講師またはチームで以下を確認してください。

- 2時間で終わる範囲になっているか
- 公式情報を言い換えすぎていないか
- 架空の実績や数字が混ざっていないか
- コンポーネント分割が細かすぎないか
- テストが実装内容に対応しているか

### ステップ3: ページの最小構成を実装する

Claude Codeに依頼:

```text
先ほどの設計に沿って、/ndb-renewal ページを実装してください。

まずは以下のセクションを表示できる最小構成にしてください。

- ヒーロー
- サービス紹介
- 会社の特徴
- 採用向けメッセージ
- News
- お問い合わせ導線

既存サイトのソースコードはコピーせず、公開情報を参考にした研修用モックとして作成してください。
レスポンシブ対応し、コンポーネントを適切に分割してください。
```

### ステップ4: 表示を確認して調整する

以下を確認してください。

- `npm run dev` で `/ndb-renewal` が表示できる
- PC幅でセクションの流れが自然
- スマートフォン幅でテキストやボタンがはみ出さない
- CTAの遷移先が不自然ではない
- 会社情報として誤解を生む文言がない

### ステップ5: テストを追加する

Claude Codeに依頼:

```text
今回追加した /ndb-renewal ページに対して、Vitestでテストを追加してください。

テストしたいこと:
- ページコンポーネントが描画できる
- ヒーローの見出しが表示される
- サービス3件が表示される
- 採用向けセクションが表示される
- お問い合わせ導線が表示される

既存のVitest設定に合わせてください。
追加依存が必要な場合は、理由を説明してから提案してください。
```

### ステップ6: レビューSkill / Sub-agentを使う

実装後、前回作ったレビューSkillまたはSub-agentで差分を確認してください。

Claude Codeに依頼:

```text
今回の変更差分をレビューしてください。

観点:
- Next.js / Reactとして自然な設計か
- コンポーネント分割は適切か
- 文言が会社情報として不自然ではないか
- レスポンシブ対応に問題がないか
- テストが不足していないか
- ジュニアエンジニアが保守しやすいか

重要度 High / Medium / Low に分けて指摘してください。
まだ修正はしないでください。
```

レビュー後、重要度Highの指摘だけを先に修正してください。
